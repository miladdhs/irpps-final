# دستورات مدیریت رویدادها

## افزودن رویدادهای جدید

برای افزودن 3 رویداد جدید به دیتابیس، از دستور زیر استفاده کنید:

### در Docker (توصیه می‌شود):

```bash
docker-compose exec backend python manage.py add_new_events
```

یا اگر container در حال اجرا است:

```bash
docker exec -it irpps-backend python manage.py add_new_events
```

### در محیط محلی (با دیتابیس آنلاین):

اگر دیتابیس آنلاین در دسترس است، می‌توانید مستقیماً از محیط محلی اجرا کنید:

```bash
cd backend
python manage.py add_new_events
```

**نکته**: مطمئن شوید که متغیرهای محیطی دیتابیس (DB_HOST, DB_USER, DB_PASSWORD, DB_NAME) به درستی تنظیم شده‌اند.

## رویدادهای اضافه شده

این دستور 3 رویداد زیر را به دیتابیس اضافه می‌کند:

1. **ورکشاپ عملی POCUS** - بهمن 1404
2. **کنفرانس علمی یک روزه: بیماری شایع تنفسی کودکان** - دی 1404
3. **چهارمین کنگره بین‌المللی ریه و اختلالات تنفسی خواب کودکان** - دی 1404

## تصاویر

تصاویر رویدادها به صورت خودکار از `frontend/public/Content/lAST/` به `backend/media/events/covers/` کپی می‌شوند.

## نکات

- دستور به صورت خودکار بررسی می‌کند که آیا رویداد با همان slug قبلاً وجود دارد یا نه
- اگر رویداد قبلاً وجود داشته باشد، از افزودن مجدد آن صرف‌نظر می‌شود
- برای ایجاد رویدادها، نیاز به یک کاربر staff در دیتابیس است
